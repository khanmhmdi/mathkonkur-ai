## Prerequisites

Before starting, ensure you have:

- **GitHub account** (https://github.com)
- **Git installed locally**
- **GapGPT API key** (get from https://api.gapgpt.app)
- **Node.js 20+ installed** locally (for testing)

## GitHub Repository Setup

### Step 1: Initialize Git Repository

```bash
# Navigate to project root
cd mathkonkur-ai

# Initialize git (if not already initialized)
git init

# Add all files
git add .

# Commit files
git commit -m "Initial commit with full application"

# Add remote repository (replace with your repo URL)
git remote add origin https://github.com/YOUR_USERNAME/mathkonkur-ai.git

# Push to main branch
git branch -M main
git push -u origin main
```

### Step 2: Create Develop Branch

```bash
# Create and switch to develop branch
git checkout -b develop

# Push develop branch
git push -u origin develop
```

## Environment Secrets Setup

### Required Secrets

You need to configure these secrets in your GitHub repository settings:

| Secret Name | Description | How to Get |
|-------------|-------------|------------|
| `VERCEL_TOKEN` | Vercel deployment token | https://vercel.com/account/tokens |
| `VERCEL_ORG_ID` | Vercel organization ID | From Vercel project settings |
| `VERCEL_PROJECT_ID` | Vercel project ID | From Vercel project settings |
| `RAILWAY_TOKEN` | Railway deployment token | https://railway.app/account/tokens |
| `GAPGPT_API_KEY` | GapGPT API key | https://api.gapgpt.app |
| `JWT_SECRET` | Random 32+ char string | `openssl rand -hex 32` |
| `JWT_REFRESH_SECRET` | Different 32+ char string | `openssl rand -hex 32` |

### Adding Secrets to GitHub

1. Go to your GitHub repository
2. Click **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Add each secret with its value

### Platform-Specific Environment Variables

**For Vercel (Frontend):**
- `VITE_GAPGPT_API_KEY` - Your GapGPT API key
- `VITE_API_URL` - Your backend URL (e.g., `https://mathkonkur-api.up.railway.app/api`)

**For Railway (Backend):**
- `NODE_ENV` = `production`
- `PORT` = `4000`
- `DATABASE_URL` - Auto-generated by Railway PostgreSQL
- `JWT_SECRET` - Your generated secret
- `JWT_REFRESH_SECRET` - Your generated refresh secret
- `GAPGPT_API_KEY` - Your GapGPT API key
- `FRONTEND_URL` - Your Vercel frontend URL

## Deployment Options

### Option A: Vercel + Railway (Recommended)

This is the recommended setup for optimal performance and cost.

#### 1. Deploy Frontend to Vercel

**Using Vercel Dashboard:**

1. Go to https://vercel.com/new
2. Import your GitHub repository
3. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `./` (root)
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Add Environment Variables:
   - `VITE_GAPGPT_API_KEY` = your GapGPT key
   - `VITE_API_URL` = placeholder (update after backend deploy)
5. Click **Deploy**

**Using Vercel CLI:**

```bash
# Install Vercel CLI
npm i -g vercel

# Login
vercel login

# Deploy
vercel --prod
```

#### 2. Deploy Backend to Railway

**Using Railway Dashboard:**

1. Go to https://railway.app/new
2. Select **Deploy from GitHub repo**
3. Choose your repository
4. Add **New** → **Database** → **Add PostgreSQL**
5. Configure environment variables in Railway dashboard:
   ```
   NODE_ENV=production
   JWT_SECRET=<your-generated-secret>
   JWT_REFRESH_SECRET=<your-generated-refresh-secret>
   GAPGPT_API_KEY=<your-gapgpt-key>
   FRONTEND_URL=<your-vercel-url>
   ```
6. Deploy will happen automatically

**Using Railway CLI:**

```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link project
cd backend
railway link

# Deploy
railway up
```

#### 3. Update Frontend with Backend URL

After backend is deployed:

1. Get your Railway backend URL (e.g., `https://mathkonkur-api.up.railway.app`)
2. Go to Vercel dashboard → Project → Settings → Environment Variables
3. Update `VITE_API_URL` to `https://your-backend-url.railway.app/api`
4. Redeploy frontend

### Option B: Render (Alternative)

Single-platform deployment with built-in PostgreSQL.

#### Using Render Dashboard

1. Go to https://dashboard.render.com/blueprints
2. Click **New Blueprint Instance**
3. Connect your GitHub repository
4. Render will read `render.yaml` and create:
   - Frontend static site
   - Backend web service
   - PostgreSQL database
5. Add these environment variables manually:
   - `GAPGPT_API_KEY`
   - Update `FRONTEND_URL` and `VITE_API_URL` after first deploy

#### Using Render CLI

```bash
# Install Render CLI (optional)
curl -fsSL https://raw.githubusercontent.com/render-oss/render-cli/main/install.sh | bash

# Deploy from blueprint
render blueprint launch
```

### Option C: Docker Self-Hosted

Deploy using Docker on your own server or cloud VPS.

#### Using Docker Compose

```bash
# Clone repository
git clone https://github.com/YOUR_USERNAME/mathkonkur-ai.git
cd mathkonkur-ai

# Create environment file
cp .env.example .env
cp backend/.env.example backend/.env

# Edit .env files with your production values
# - VITE_GAPGPT_API_KEY
# - DATABASE_URL
# - JWT_SECRET
# - JWT_REFRESH_SECRET

# Start all services
docker-compose up -d

# Run database migrations
docker-compose exec backend npx prisma migrate deploy

# Check logs
docker-compose logs -f
```

#### Manual Docker Deployment

```bash
# Build frontend image
docker build -t mathkonkur-frontend .

# Build backend image
cd backend
docker build -t mathkonkur-backend .

# Run containers (adjust environment variables)
docker run -d -p 3000:80 --name frontend mathkonkur-frontend

docker run -d -p 4000:4000 \
  -e NODE_ENV=production \
  -e DATABASE_URL=postgresql://... \
  -e JWT_SECRET=... \
  -e GAPGPT_API_KEY=... \
  --name backend mathkonkur-backend
```

## Post-Deployment

### 1. Database Migration

After first deployment, run migrations:

**Railway:**
```bash
cd backend
railway run npx prisma migrate deploy
```

**Render:**
- Automatic via `render.yaml` startCommand

**Docker:**
```bash
docker-compose exec backend npx prisma migrate deploy
```

### 2. Seed Database (Optional)

```bash
# Railway
railway run npx prisma db seed

# Docker
docker-compose exec backend npx prisma db seed
```

### 3. Health Check

Verify deployment:

```bash
# Frontend
curl https://your-frontend-url.vercel.app

# Backend
curl https://your-backend-url.railway.app/health
```

### 4. Configure Custom Domain (Optional)

**Vercel:**
1. Dashboard → Project → Settings → Domains
2. Add your domain

**Railway:**
1. Dashboard → Service → Settings → Domains
2. Add custom domain

## Troubleshooting

### Common Issues

#### 1. Frontend can't connect to backend

**Check:**
- `VITE_API_URL` is set correctly (must include `/api` suffix)
- Backend CORS allows frontend URL
- No trailing slash mismatch

**Fix:**
```env
# Correct
VITE_API_URL=https://api.example.com/api

# Wrong
VITE_API_URL=https://api.example.com/api/  # trailing slash
VITE_API_URL=https://api.example.com         # missing /api
```

#### 2. Database connection errors

**Check:**
- `DATABASE_URL` format: `postgresql://user:pass@host:5432/db?schema=public`
- Database is running and accessible
- SSL mode for cloud databases (add `?sslmode=require`)

#### 3. JWT errors

**Check:**
- `JWT_SECRET` and `JWT_REFRESH_SECRET` are at least 32 characters
- Secrets match between environment variables

#### 4. Build fails

**Check:**
- Node version is 20+
- All dependencies installed
- TypeScript compiles locally: `npm run build`

#### 5. Prisma errors

**Fix:**
```bash
# Regenerate client
npx prisma generate

# Reset migrations (local only)
npx prisma migrate reset
```

## CI/CD Workflow

The GitHub Actions workflow (`/.github/workflows/ci-cd.yml`) automatically:

1. **On Pull Request to develop/main:**
   - Runs all tests
   - Builds application
   - Runs security scans

2. **On Push to develop:**
   - Deploys frontend to Vercel (Preview)
   - Deploys backend to Railway (Staging)

3. **On Push to main:**
   - Deploys frontend to Vercel (Production)
   - Deploys backend to Railway (Production)

### Manual Deployment

To deploy manually (skip CI):

```bash
# Frontend
vercel --prod --token=YOUR_TOKEN

# Backend
railway up --token=YOUR_TOKEN
```
